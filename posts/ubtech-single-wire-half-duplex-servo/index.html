            🔍 优必选舵机（单线半双工）#

            这类优必选舵机不是普通 PWM 舵机，而是单线半双工串口总线舵机。

            🔌 接口识别#

            这类舵机通常是一对 3Pin 接口，但实际有用的核心连接仍然是三类：

            • GND：地
            • VCC：舵机电源 -> 8V
            • SIG：单线总线信号

            🤔 单线半双工通信板#

            单线半双工总线的本质，是把主机的 TX 和 RX 通过一段隔离或门控电路接到同一根 BUS 线上。 单线半双工通信板 单线半双工通信板

            通信板需要完成两件事：

            1. 主机发送时，让 TX 能驱动总线。
            2. 主机接收时，不让 TX 直接回灌到 RX。

            📋 协议帧格式#

            优必选舵机基础帧格式如下：

            +------+------+------+------+------+------+------+------+--------+------+
| FA   | AF   | ID   | CMD  | P1   | P2   | P3   | P4   | CHECK  | ED   |
+------+------+------+------+------+------+------+------+--------+------+
| 帧头 | 帧头  | 地址 | 指令 | 参数1 | 参数2| 参数3 | 参数4 | 校验和 | 结尾 |
+------+------+------+------+------+------+------+------+--------+------+

            💡 提示：帧格式固定为 10 字节。发送前记得计算校验和，公式为 CHECK = (ID + CMD + P1 + P2 + P3 + P4) & 0xFF。

            字段速查表#

            字段含义典型取值
            0xFA帧头 1固定
            0xAF帧头 2固定
            ID舵机地址0x00、0x01、…、0xFF
            CMD命令字0x01运动、0x02读角度、0xCD改ID、0xD2设偏移、0xD4读偏移
            P1~P4参数区依命令不同而变化
            CHECK校验低 8 位累加和
            0xED结束字节固定

            💡 提示：舵机地址0x00为广播，可以操纵所有舵机。

            校验规则可以按参考代码理解为：

            1

            CHECK = (ID + CMD + P1 + P2 + P3 + P4) & 0xFF

            📡 常用指令#

            以下按运动控制、信息查询、配置三类，逐条列出每条命令的发送帧与舵机反馈。

            运动控制类#

            0x01 指定时间运动#

            用途：让舵机在指定时间内运动到目标角度。

            参数说明：

            参数含义取值范围单位
            P1目标角度0–240度（°）
            P2运动时间0–25520ms
            P3保留0x00—
            P4保留0x00—
            • 目标角度最大有效值为 240 度，即0xF0
            • 运动时间为 0 时，舵机全速转动
            • 从舵机收到命令开始计时，运动时间未结束之前，舵机不再响应新的 0x01 命令
            • 到位之后，如果外力导致角度偏移，舵机会强制调整保持角度

            发送示例：舵机 ID=1，转到 90 度，运动时间 20（约 400ms）

            1

            FA AF 01 01 5A 14 00 00 70 ED

            逐字段解释：

            • FA AF：帧头
            • 01：舵机 ID = 1
            • 01：运动命令
            • 5A：目标角度 = 0x5A = 90 度
            • 14：运动时间 = 0x14 = 20（约 400ms）
            • 00 00：保留
            • 70：校验 = (01+01+5A+14+00+00) & 0xFF = 0x70
            • ED：帧尾

            反馈：

            • 成功/正确：仅回复 1 字节 0xAA + ID
            • 失败/错误：不返回任何数据

            0x01 + 0xFF 强制中止转动#

            用途：不管舵机之前是否已转动到指定位置，收到此命令后立刻停止转动，同时舵机失电，仅靠齿轮组阻尼维持位置。

            这是 0x01 命令的特殊用法：当 P1 = 0xFF 时触发强制中止。

            参数说明：

            参数含义取值
            P1中止标志0xFF（固定）
            P2—0x00
            P3—0x00
            P4—0x00

            发送示例：中止舵机 ID=1 的当前运动

            1

            FA AF 01 01 FF 00 00 00 01 ED

            逐字段解释：

            • 01：舵机 ID = 1
            • 01：运动命令
            • FF：P1 = 0xFF，触发强制中止
            • 00 00 00：其余参数补零
            • 01：校验 = (01+01+FF+00+00+00) & 0xFF = 0x01
            • ED：帧尾

            反馈：

            • 成功/正确：仅回复 1 字节 0xAA + 舵机 ID
            • 失败/错误：不返回任何数据

            信息查询类#

            0x02 角度回读#

            用途：读取舵机当前的目标角度和实际角度。舵机返回角度数据后失电，仅靠齿轮组阻尼维持位置。

            参数说明：

            参数含义取值
            P1–P4全部补零0x00 0x00 0x00 0x00

            发送示例：读取舵机 ID=1 的角度

            1

            FA AF 01 02 00 00 00 00 03 ED

            逐字段解释：

            • 01：舵机 ID = 1
            • 02：角度回读命令
            • 00 00 00 00：参数全部补零
            • 03：校验 = (01+02+00+00+00+00) & 0xFF = 0x03
            • ED：帧尾

            反馈：

            舵机返回标准 10 字节帧，包含目标角度和实际角度：

            1

            FA AF ID 02 目标角度高 目标角度低 实际角度高 实际角度低 CHECK ED

            示例回应（假设目标角度 90 度，实际角度 85 度）：

            1

            FA AF 01 02 00 5A 00 55 B2 ED
            • 00 5A：目标角度 = 0x005A = 90 度
            • 00 55：实际角度 = 0x0055 = 85 度
            • 两个值不相同，说明舵机还未完全到位或存在控制误差
            • B2：校验 = (01+02+00+5A+00+55) & 0xFF = 0xB2

            配置类#

            0xCD 修改舵机 ID#

            用途：修改舵机的通信地址。此时尽量不要使用广播模式，避免多个舵机都占默认地址。

            参数说明：

            参数含义取值
            P1保留0x00
            P2新 ID1–240
            P3保留0x00
            P4保留0x00

            发送示例：将舵机 ID 从 1 改为 2

            1

            FA AF 01 CD 00 02 00 00 D0 ED

            逐字段解释：

            • 01：当前舵机 ID = 1
            • CD：修改 ID 命令
            • 00：保留
            • 02：新 ID = 2
            • 00 00：保留
            • D0：校验 = (01+CD+00+02+00+00) & 0xFF = 0xD0
            • ED：帧尾

            反馈：

            舵机返回标准帧，其中 Byte2（ID 字段）已经是修改后的新 ID：

            1

            FA AF 02 CD 00 00 00 00 CF ED
            • 02：ID 已变为 2（修改后的新 ID）
            • CF：校验 = (02+CD+00+00+00+00) & 0xFF = 0xCF

            0xD2 设置角度偏移量#

            用途：设置舵机的角度零点偏移校正值。面向舵盘观察，正值顺时针偏移，负值逆时针偏移。

            参数说明：

            参数含义取值范围单位
            P1保留0x00°
            P2保留0x00°
            P3偏移量高字节见下方说明°
            P4偏移量低字节见下方说明°

            偏移量为 16 位有符号整数：

            • 调整范围：-30° 到 +30°
            • 舵机重新上电后会将设定值清零
            • 负整数使用补码编码

            常见偏移值编码对照：

            偏移角度原始值（×3）高字节 P3低字节 P4
            +30 度+900x000x5A
            +3 度+90x000x09
            0 度00x000x00
            -5 度-150xFF0xF1
            -30 度-900xFF0xA6

            发送示例 1：舵机 ID=1，设置 +3 度偏移（原始值 = 9）

            1

            FA AF 01 D2 00 00 00 09 DC ED
            • 00 09：偏移量 = 9，即 +3 度
            • DC：校验 = (01+D2+00+00+00+09) & 0xFF = 0xDC

            发送示例 2：舵机 ID=1，设置 -5 度偏移（原始值 = -15，补码 0xFFF1）

            1

            FA AF 01 D2 00 00 FF F1 C3 ED
            • FF F1：偏移量 = -15（补码），即 -5 度
            • C3：校验 = (01+D2+00+00+FF+F1) & 0xFF = 0xC3

            0xD4 读取角度偏移量#

            用途：读取当前设置的角度偏移校正值。

            参数说明：

            参数含义取值
            P1–P4全部补零0x00 0x00 0x00 0x00

            发送示例：读取舵机 ID=1 的偏移量

            1

            FA AF 01 D4 00 00 00 00 D5 ED
            • D5：校验 = (01+D4+00+00+00+00) & 0xFF = 0xD5

            反馈：

            舵机返回标准帧，参数区携带偏移校正值：

            1

            FA AF 01 D4 00 00 00 09 DE ED
            • 00 09：偏移量 = 9，即 +3 度（顺时针）
            • DE：校验 = (01+D4+00+00+00+09) & 0xFF = 0xDE

            指令总览#

            CMD名称分类反馈方式
            0x01指定时间运动运动控制成功返回 0xAA + ID，失败无返回
            0x01+0xFF强制中止转动运动控制无返回，舵机立即失电
            0x02角度回读信息查询返回目标角度 + 实际角度
            0xCD修改舵机 ID配置返回标准帧，ID 字段为新 ID
            0xD2设置角度偏移量配置成功返回 0xAA，失败无返回
            0xD4读取角度偏移量配置返回偏移校正值

            💡 提示：成功返回 0xAA，失败返回 0xEE。

            ⏱ 运动时间与锁定机制#

            参考项目里的控制代码把时间参数做了这样的换算：

            1

            cmd[5] = int(motion_time_ms * speed / (100 * 20)) & 0xFF;

            这说明在最基础用法里，可以先把 P2 理解为一个与时间相关的缩放值，按 20ms 为一个单位 去试比较容易上手。

            运动时间#

            • 单位：约 20ms
            • 取值范围：0–255（单字节）或 0–65535（双字节扩展协议）
            • 值为 0 时：舵机全速转动
            • 示例：
              • P2 = 0x14（20）→ 约 400ms
              • P2 = 0x32（50）→ 约 1000ms = 1 秒

            锁定机制#

            根据 PDF 原文描述，从舵机收到 0x01 命令开始计时，运动时间未结束之前，舵机不再响应新的 0x01 命令。这相当于一种“锁定”效果：

            • 舵机在运动期间不会被打断
            • 到位之后，如果外力导致角度偏移，舵机会强制调整保持角度
            • 这种锁定是自动的，不需要额外设置参数

            💻 ESP32 最小控制示例#

            下面给一个最小示例，单片机采用 ESP32C3，向总线上所有舵机发送广播运动命令：

            #include <HardwareSerial.h>
HardwareSerial MySerial1(1);  // 使用串口1
// 这是一条广播命令，让总线上所有舵机全速转到 128 度位置。
const byte sendData[] = {0xFA, 0xAF, 0x00, 0x01, 0x80, 0x00, 0x00, 0x00, 0x81, 0xED};
void setup() {
  Serial.begin(115200);                          // 调试串口
  MySerial1.begin(115200, SERIAL_8N1, 21, 20);   // 发送串口
}
void loop() {
  MySerial1.write(sendData, sizeof(sendData));   // 发送数据
  delay(100); 
}

            📝 总结#

            这类优必选拆机舵机最核心的知识点，其实就三件事：

            1. 它不是 PWM 舵机，而是单线半双工串口舵机。
            2. 要想稳定控制，必须先做好 TX/RX 与 BUS 的隔离通信板。
            3. 控制上先掌握 0x02 读角度和 0x01 运动命令，就已经能完成大部分入门实验。
