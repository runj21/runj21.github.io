                      📖 第一章：波士顿房价预测#

                      1.1 项目简介#

                      以经典的 波士顿房价数据集 为起点，探索同一个回归任务在传统机器学习和深度学习两种范式下的表现差异。

                      💡 一句话总结：13 个数值特征 → 预测房价中位数（MEDV，单位：千美元），分别用线性回归、随机森林、3 层全连接神经网络实现，对比 MAE、R² 等指标。

                      数据集包含 506 条样本，13 个特征：

                      特征含义
                      CRIM城镇人均犯罪率
                      ZN超过 25000 平方英尺的住宅用地比例
                      INDUS非零售商业用地比例
                      CHAS是否临近查尔斯河（1=是，0=否）
                      NOX氮氧化物浓度
                      RM平均每栋住宅的房间数
                      AGE1940 年之前建造的老房子比例
                      DIS到波士顿五个就业中心的加权距离
                      RAD到高速公路的可达性指数
                      TAX每 10000 美元的房产税率
                      PTRATIO城镇师生比
                      B(1000(Bk - 0.63)^2)，其中 Bk 是黑人比例
                      LSTAT低收入人口比例

                      目标变量：MEDV — 自住房屋的中位数价格（千美元）。

                      📖 第二章：MNIST 手写数字识别#

                      2.1 项目简介#

                      MNIST（Modified National Institute of Standards and Technology）是深度学习界的 “Hello World”——一个包含 0~9 十类手写数字的灰度图片数据集。本章用 PyTorch 搭建卷积神经网络（CNN），覆盖了数据加载 → 模型设计 → 训练 → 推理 → 量化部署的完整流程。

                      💡 一句话总结：28×28 的灰度手写数字图片 → 用 LeNet-5 风格 CNN 识别成 0~9 的数字，准确率 99%+，最后将模型量化为 INT8，大小缩小约 4 倍。

                      维度详情
                      任务类型图像分类（10 类）
                      输入尺寸1×28×28（单通道灰度图）
                      模型4 层卷积 + 2 层全连接
                      测试准确率99%+
                      模型大小FP32: ~1.69 MB → INT8: ~0.43 MB

                      import pandas as pd
import io
from PIL import Image
from torch.utils.data import Dataset


class MNISTParquetDataset(Dataset):
    def __init__(self, parquet_path, transform=None):
        self.df = pd.read_parquet(parquet_path)
        self.transform = transform

    def __len__(self):
        return len(self.df)

    def __getitem__(self, idx):
        row = self.df.iloc[idx]
        image_dict = row['image']
        png_bytes = image_dict['bytes']
        img = Image.open(io.BytesIO(png_bytes)).convert('L')
        label = int(row['label'])

        if self.transform:
            img = self.transform(img)
        return img, label

                      💡 这种设计的好处是：数据以标准图片格式存储，不依赖专有二进制格式，方便在非机器学习工具中查看和验证。

                      import torch.nn as nn
import torch.nn.functional as F


class MNISTCNN(nn.Module):
    def __init__(self):
        super(MNISTCNN, self).__init__()
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.conv3 = nn.Conv2d(64, 64, kernel_size=3, padding=1)
        self.conv4 = nn.Conv2d(64, 64, kernel_size=3, padding=1)
        self.pool = nn.MaxPool2d(2, 2)
        self.dropout1 = nn.Dropout2d(0.25)
        self.dropout2 = nn.Dropout(0.5)

        self.fc1 = nn.Linear(64 * 7 * 7, 128)
        self.fc2 = nn.Linear(128, 10)

    def forward(self, x):
        x = F.relu(self.conv1(x))
        x = F.relu(self.conv2(x))
        x = self.pool(x)

        x = F.relu(self.conv3(x))
        x = F.relu(self.conv4(x))
        x = self.pool(x)
        x = self.dropout1(x)

        x = x.view(x.size(0), -1)   # 展平
        x = F.relu(self.fc1(x))
        x = self.dropout2(x)
        x = self.fc2(x)
        return x

                      设计要点#

                      • padding=1 → 保持卷积后特征图尺寸不变，让池化层负责缩小
                      • 两次 MaxPool → 28×28 → 14×14 → 7×7，最终特征图大小为 7×7
                      • 两个 Dropout（0.25 和 0.5）→ 防止过拟合，提升泛化能力
                      • 模型参数量约 42 万，轻量且高效

                      def predict_image(model, image_tensor, device):
    model.eval()
    with torch.no_grad():
        image_tensor = image_tensor.unsqueeze(0).to(device)
        output = model(image_tensor)
        probs = F.softmax(output, dim=1)
        pred = output.argmax(dim=1).item()
        confidence = probs[0][pred].item()
    return pred, confidence

                      2.6 模型量化：从 FP32 到 INT8#

                      为什么要量化？#

                      深度学习模型默认使用 FP32（32 位浮点数），每个参数占 4 字节。量化将参数转为 INT8（8 位整数），每个参数仅占 1 字节——模型大小缩小约 4 倍。

                      💡 量化特别适合 CPU 部署场景，模型更小、加载更快、缓存命中率更高，而准确率几乎不受影响。

                      动态量化实现#

                      使用 PyTorch 的动态量化，只量化全连接层，卷积层保持 FP32：

                      def quantize_dynamic_model(model):
    quantized_model = torch.quantization.quantize_dynamic(
        model,
        {nn.Linear},       # 只量化全连接层
        dtype=torch.qint8  # 目标精度 INT8
    )
    return quantized_model

                      def load_model(precision, model_path=None):
    if precision == 'fp32':
        model = YOLO(DEFAULT_MODEL)
        return model, 'FP32', False

    elif precision == 'fp16':
        model = YOLO(DEFAULT_MODEL)
        model.model.half()
        return model, 'FP16', False

    elif precision == 'fp8':
        engine_path = os.path.join(WEIGHTS_DIR, 'yolo26n_fp8.engine')
        model = YOLO(engine_path)
        return model, 'FP8-TRT', False

    elif precision == 'nvfp4':
        from torchao.quantization import quantize_
        from torchao.prototype.mx_formats import NVFP4WeightOnlyConfig

        base_model = YOLO(DEFAULT_MODEL)
        pytorch_model = base_model.model.eval().cuda()
        quantize_(pytorch_model, NVFP4WeightOnlyConfig())
        return pytorch_model, 'NVFP4', True

                      def build_trt_engine(precision, force=False):
    # Step 1: 导出 FP32 ONNX
    model = YOLO(MODEL_PT)
    onnx_fp32 = model.export(format='onnx', imgsz=640, opset=19)

    # Step 2: ModelOpt 量化
    from modelopt.onnx.quantization import quantize
    calib_data = make_calib_data(n=32)
    quantize(
        onnx_path=onnx_fp32,
        quantize_mode=precision,  # fp8 or int8
        calibration_data={'images': calib_data.cpu().numpy()},
    )

    # Step 3: TRT 构建 engine
    import tensorrt as trt
    logger = trt.Logger(trt.Logger.WARNING)
    builder = trt.Builder(logger)
    network = builder.create_network()
    parser = trt.OnnxParser(network, logger)
    parser.parse_from_file(out_onnx)
    config = builder.create_builder_config()
    config.set_memory_pool_limit(trt.MemoryPoolType.WORKSPACE, 4 * 1 << 30)
    engine = builder.build_serialized_network(network, config)

                      3.5 量化方案对比#

                      精度后端模型大小推理速度适用场景
                      FP32TRT~10.8 MB基准精度优先
                      FP16TRT~5.5 MB~2x通用 GPU 推理
                      FP8TRT~5.2 MB~2.5x推荐，速度最快
                      INT8TRT~28.8 MB~2.4x需要校准数据
                      NVFP4-WOPyTorch–~2.2x实验性，需 torchao
                      NVFP4-DynPyTorch–~2.3x实验性，动态量化

                      💡 FP8 是目前性价比最高的方案：模型大小和 FP16 几乎一样，但速度最快。INT8 的 engine 文件较大是因为包含了额外的量化参数。
