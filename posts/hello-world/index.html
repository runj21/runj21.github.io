            👋 你好，世界#

            欢迎来到我的个人博客！这是使用 Hugo + PaperMod 搭建的。

            折腾了一圈之后选了这套方案，原因很简单：快、轻量、好看。Hugo 构建速度毫秒级，一个二进制文件就能跑，PaperMod 主题简洁又支持暗色模式。

            下面把整个搭建过程记录下来，如果你也想搭建一个自己的博客，跟着走就行。

            📋 阅读提示：这篇文章面向零基础用户，步骤尽量详细。如果你已经有 Git 或 Hugo 经验，可以快速扫过对应小节。

            📦 第一步：环境准备#

            🌐 安装 Git#

            Hugo 站点需要 Git 来管理版本和部署。先去下载安装：

            👉 Git 官网下载

            安装时一路默认即可，装完打开终端验证：

            1

            git --version

            看到版本号说明安装成功。

            ⚙️ 安装 Hugo#

            前往 GitHub Releases 页面下载：

            👉 github.com/gohugoio/hugo/releases

            根据你的系统选择对应版本（以 Windows 为例）：

            文件名说明
            hugo_extended_xxx_windows-amd64.zip推荐，扩展版，支持 SCSS/SASS
            hugo_xxx_windows-amd64.zip标准版

            下载后解压，把 hugo.exe 放在一个固定目录，比如：

            1
2

            C:\hugo\
└── hugo.exe

            ✅ 验证安装#

            在终端中进入 hugo.exe 所在目录，运行：

            1
2

            cd C:\hugo
.\hugo.exe version

            看到类似输出说明安装成功：

            1

            hugo v0.146.4-xxxxxx windows/amd64 BuildDate=xxxx

            💡 提示：如果想全局使用 hugo 命令，可以把 C:\hugo 添加到系统环境变量 PATH 中。不过不加也没关系，后面用相对路径调用即可。

            🚀 第五步：部署上线#

            方案选择#

            Hugo 静态博客有很多部署方式：GitHub Pages、Vercel、Netlify、Cloudflare Pages 等。我选的是 GitHub Pages + 双仓库手动推送，原因是：

            • 免费、无限带宽
            • 支持自定义域名
            • 源码可以放在私有仓库，不公开

            仓库结构#

            需要两个 GitHub 仓库：

            仓库可见性用途
            my-blog私有存放 Hugo 源码（Markdown、配置文件等）
            username.github.io公开存放构建后的静态文件（HTML、CSS、图片）

            💡 username.github.io 是 GitHub Pages 的特殊仓库名，会自动部署到 https://username.github.io/。

            创建 GitHub 仓库#

            1. 在 GitHub 上新建仓库 my-blog（勾选 Private）
            2. 在本地站点目录初始化 Git 并推送：
            1
2
3
4
5
6

            cd MyBlog
git init
git remote add origin https://github.com/你的用户名/my-blog.git
git add -A
git commit -m "初始提交"
git push -u origin main
            1. 再创建一个公开仓库 你的用户名.github.io（勾选 Public）

            配置 GitHub Pages#

            进入 username.github.io 仓库的 Settings → Pages：

            • Source 选择 Deploy from a branch
            • Branch 选择 main，文件夹选 / (root)
            • 点击 Save

            创建 Personal Access Token (PAT)#

            部署脚本需要权限才能推送到 username.github.io，所以需要创建一个 PAT：

            1. 访问 GitHub Settings → Developer settings → Personal access tokens → Fine-grained tokens
            2. 点击 Generate new token
            3. 设置：
              • Token name：随意，比如 hugo-deploy
              • Expiration：选择过期时间
              • Repository access：选择 username.github.io 仓库
              • Permissions：添加 Contents: Read and write
            4. 生成后复制 Token（只显示一次！）

            # ============================================
# Hugo 博客一键部署脚本
# 用法: .\deploy.ps1 "提交信息"
# 流程: 提交源码 → 构建站点 → 部署到 GitHub Pages
# ============================================

param([string]$Message = "Update blog content")

$ErrorActionPreference = "Stop"
$SiteRoot = $PSScriptRoot
$HugoRoot = Split-Path $SiteRoot -Parent

# 1. 检查 Git 状态
Set-Location $SiteRoot
$changes = git status --porcelain
if ([string]::IsNullOrWhiteSpace($changes)) {
    Write-Host "未检测到任何变更，无需部署。" -ForegroundColor Yellow
    exit 0
}

# 2. 提交源码
git add -A
git commit -m "$Message"

# 3. 推送源码到 my-blog 仓库（私有）
git push origin main

# 4. 构建 Hugo 站点（每次清空 public 避免旧文件残留）
if (Test-Path "public") { Remove-Item -Recurse -Force "public" }
& "$HugoRoot\hugo.exe" --minify --buildFuture --baseURL "https://username.github.io/"

# 5. 部署到 username.github.io 仓库（公开）
$DeployToken = $env:HUGO_DEPLOY_TOKEN
if (-not $DeployToken) {
    Write-Error "请先设置环境变量 HUGO_DEPLOY_TOKEN，再运行部署脚本。"
    exit 1
}
$DeployRemote = "https://你的用户名:$DeployToken@github.com/你的用户名/username.github.io.git"

Set-Location "public"
if (-not (Test-Path ".git")) {
    # 首次初始化
    git init
    git remote add origin $DeployRemote
    git fetch origin main --depth=1
    git checkout -b main
} else {
    # 后续更新：拉取远程状态，只推送变更
    git remote set-url origin $DeployRemote
    git fetch origin main --depth=1
    git reset --soft origin/main
}

# 删除 .github 目录（避免 PAT 权限范围报错）
Remove-Item -Recurse -Force ".github" -ErrorAction SilentlyContinue

# 提交并强制推送
git add -A
$timestamp = Get-Date -Format "yyyy-MM-dd HH:mm:ss"
git commit -m "部署博客 - $timestamp"
git push --force origin main

# 6. 返回源码目录
Set-Location $SiteRoot
Write-Host "部署完成！约 1~2 分钟后生效。" -ForegroundColor Green
Write-Host "源码仓库：https://github.com/你的用户名/my-blog"
Write-Host "Pages 地址：https://username.github.io"

            🛡️ 安全提示：PAT 通过环境变量 HUGO_DEPLOY_TOKEN 传入，不要直接写进脚本。运行前在当前终端设置：

            1

            $env:HUGO_DEPLOY_TOKEN = "ghp_xxxxxxxx"

            另外建议把 deploy.ps1 加入 .gitignore，避免仓库泄露用户名等细节：

            1

            deploy.ps1

            脚本工作流程#

            一键部署#

            以后每次写完文章，先确保已设置 HUGO_DEPLOY_TOKEN，然后运行：

            1
2

            $env:HUGO_DEPLOY_TOKEN = "ghp_xxxxxxxx"
.\deploy.ps1 "新增文章：xxx"

            脚本会自动完成：提交源码 → 构建站点 → 推送到 Pages 仓库。

            访问你的博客#

            部署后 1~2 分钟生效，访问地址：

            • 🏠 主页：https://username.github.io/
            • 📝 文章：https://username.github.io/posts/文章目录名/
            • 📚 归档：https://username.github.io/archives/
            • 🏷️ 标签：https://username.github.io/tags/

            ⚠️ 踩过的坑#

            搭建过程中遇到过几个问题，记录一下：

            部署后 404：public/ 目录残留了旧文件。解决办法是在 deploy.ps1 里每次构建前先清空 public/。

            中文乱码：PowerShell 终端编码不是 UTF-8。在 VSCode 右下角切换到 UTF-8 保存，或者设置终端环境变量。

            推送超时：一般是代理没开。在终端里手动设置环境变量：

            1
2

            $env:HTTP_PROXY = "http://127.0.0.1:端口号"
$env:HTTPS_PROXY = "http://127.0.0.1:端口号"

            💡 注意：不同代理软件的监听端口不同，需要查看你自己软件的设置：

            • Clash — 默认 7890
            • Clash Verge — 默认 7897
            • V2RayN — 默认 10809
            • Xray / sing-box — 默认 2080 或自定义

            在代理软件的设置页确认实际端口，替换上面的“端口号”即可。

            文章不显示：检查 draft 是不是 true，或者 date 是不是写成了未来日期。
